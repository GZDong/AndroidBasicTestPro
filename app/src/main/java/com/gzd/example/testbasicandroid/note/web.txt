1.xml解析常用的有2种形式，一种是使用Pull，特点是构建方便，但是解析麻烦，需要手工解析；
  它的流程是以字符流的方式读取数据，然后默认从第一个标签开始，一个个手动的形式往后解析提取的形式工作；
  另一种是SAX，它是构造麻烦，但是解析方便，需要手工构造handler；
  它的流程是通过构造一个handler，然后通过将标签的层次封装成了一个个方法去对应，在方法里直接可以获取该层次的标签信息
2.原生的网络请求是通过构建一个HttpUrlConnection并获取流的方式进行的，可以以http的形式进行设定和分析
3.OKHttp和原生的网络请求不同，它没有站在一个http连接的角度，而是通过封装成更加语义化的client，request，newCall，requestBody等等对象；
  另一个好处是，不需要手动开启线程，不需要手动创建回调接口,不需要手动关闭资源；
4.json解析常用2种，一种是使用JSONObject，使用方法很简单，直接用一个String构建一个JSONArray，然后便利JSONArray，
  提取出里面从0开始的JSONObject，再通过JSONObject的方式提取key对应的值，缺点是不能直接转化成数据对应的对象，需要手工植入；
  另一种是GSON解析，它不需要解析提取，它可以直接帮你把数据封装进一个对象中，当中使用到匿名的TypeToken方法去构造数据容器，是因为对于多
  对象的数据集，存在泛型的干扰，会使GSON的反射出现问题，所以一般使用这个进行数据转行
5.WebView中直接封装了Http请求的各种东西，也可以手动修改默认值
